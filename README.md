# Aphelion

> *aphelion (n.) — the point in an orbit furthest from the sun*

A glowing-ASCII space trading and combat game. Inspired by the design of classic 8-bit space sims, built with a custom rendering pipeline that draws every ship, planet, and dashboard element from text symbols passed through a multi-pass bloom shader.

The aesthetic conceit: you are receiving fragmented signals from a galaxy too far away to image directly, decoded into glowing characters.

## Status

**Pre-alpha.** The chart-to-flight game loop works end to end: explore eight procedural galaxies on a galactic chart, lock a system as a jump target, watch the hyperspace tunnel race past, arrive at the destination's sun and planet, fly around with proper six-axis controls. See `docs/roadmap.md` for what's left.

## Running

The game is a single self-contained HTML file. Open `index.html` in any modern browser — no build step, no server.

For development:

```bash
npm install
npm test                    # 78 tests across galaxy, themes, ship sim, save
npx tsx scripts/inspect.ts  # print sample systems from the canonical galaxy
```

## What works

- Procedural galaxy: 8 galaxies × 256 systems each, deterministic from a single seed triple. Every system has a name, coordinates, economy, government, tech level, population, productivity, and a 1-2 sentence flavor description generated by a recursive grammar.
- Galactic chart: glowing-ASCII map with hover info panel, click-to-lock target selection, a flyout showing the locked system's name and jump distance, and a 3-second countdown overlay before hyperspace.
- Hyperspace: tunnel effect with concentric spinning rings, accelerating into a destination flash.
- Flight: first-person cockpit view inside the player's Cobra, with WASD pitch/yaw, QE roll, Shift/Ctrl throttle. Stardust streaks past at speed-proportional rates. The destination system has a sun and a planet rendered in world space.
- Six themes: Cyan, Green Phosphor, Amber Monochrome, Cyan & Amber, Magenta Synthwave, Full Color. Persisted in localStorage.
- Save: parking spot and current galaxy persist across reloads.

## Project layout

```
src/
  types.ts                 — public type definitions
  theme.ts                 — palette system + persistence
  galaxy/
    twist.ts               — seed-twist primitive + galaxy rotation
    names.ts               — system + galaxy name generation
    generator.ts           — full galaxy assembly
    describe.ts            — recursive grammar for system descriptions
  sim/
    vec.ts                 — vector / quaternion math
    ship.ts                — flight model
    state.ts               — game state reducer
    save.ts                — save / load
  render/
    input.ts               — keyboard input layer
test/
  *.test.ts                — 78 tests across all modules
scripts/
  inspect.ts               — dev-only: print sample galaxy
index.html                 — the game (single-file app)
```

## Tech

Plain TypeScript. Vitest for tests. Canvas2D for rendering. No frameworks, no heavy dependencies. The game ships as a single inlined HTML file, well under 200 KB gzipped — a small homage to the 22 KB original that fit the entire universe in a cassette tape.

The TS source under `src/` is the canonical version of every module; `index.html` mirrors them inline so the game runs without a build step. When we move to a build pipeline, `index.html` becomes the entry point that imports from `src/`.

## Inspiration and credit

The design space this project explores was opened up by Ian Bell and David Braben's Elite (1984) and given a fresh contemporary treatment by Andrei Fomin's Effulgence RPG. Everything in this repo is original work; the galaxy generation and game systems are clean-room reimplementations written from conceptual descriptions rather than ports.

Specific debts:
- Mark Moxon's [bbcelite.com](https://elite.bbcelite.com) site, which documents the design of the original game in extraordinary depth.
- Andrei Fomin's Effulgence RPG, which proved that glowing ASCII could carry a modern game.
